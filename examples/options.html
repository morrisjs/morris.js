Most of these option was copied and pasted directly from here.
There were one or two options that were not documented.
This is just a quick reference for the chart options.

General Options
Option Default Description
dateFormat null A function that accepts millisecond timestamps and formats them for display as chart labels.
default: function (x) { return new Date(x).toString(); }
axes true Set to false to disable drawing the x and y axes.
grid true Set to false to disable drawing the horizontal grid lines.
gridLineColor '#aaa' The color that you want your gridlines to be in hexadecimal format.
gridStrokeWidth 0.5 The width of the grid lines (thickness)
gridTextSize 12 Set the point size of the axis labels
gridTextColor '#888' Set the color of the axis labels
gridTextFamily 'sans-serif' Set the font family of the axis labels
gridTextWeight 'normal' Set the font weight of the axis labels. ('bold', 'normal')
hideHover false Set to false to always show a hover legend.
Set to 'auto' to only show the hover legend when the mouse cursor is over the chart.
Set to true to never show a hover legend.
hoverCallback Provide a function on this option to generate custom hover legends.
The function will be called with the index of the row under the hover legend, the options object passed to the constructor as arguments, and a string containing the default generated hover legend content HTML.
eg:
hoverCallback: function (index, options, content) {
  var row = options.data[index];
  return "sin(" + row.x + ") = " + row.y;
}
yLabelFormat null A function that accepts y-values and formats them for display as y-axis labels.
eg: function (y) { return y.toString() + 'km'; }
xLabelAngle 0 This will tilt the the angle of the x Labels.
numLines 5 The number of gridline to draw.
padding 25 The amount of padding (top and bottom to use)
parseTime true Set to false to skip time/date parsing for X values, instead treating them as an equally-spaced series.
postUnits '' Set to a string value (eg: '%') to add a label suffix all y-labels.
preUnits '' Set to a string value (eg: '$') to add a label prefix all y-labels.
ymax 'auto' Max. bound for Y-values. Alternatively, set this to 'auto' to compute automatically, or 'auto [num]' to automatically compute and ensure that the max y-value is at least [num].
ymin 'auto 0' Min. bound for Y-values. Alternatively, set this to 'auto' to compute automatically, or 'auto [num]' to automatically compute and ensure that the min y-value is at most [num].
Hint: you can use this to create graphs with false origins.
goals [] A list of y-values to draw as horizontal 'goal' lines on the chart.
eg: goals: [1.0, -1.0]
goalStrokeWidth 1.0 Width, in pixels, of the goal lines.
goalLineColors ['#666633', '#999966', '#cc6666', '#663333'] Array of color values to use for the goal line colors. If you list fewer colors here than you have lines in goals, then the values will be cycled.
events [] A list of x-values to draw as vertical 'event' lines on the chart.
eg: events: ['2012-01-01', '2012-02-01', '2012-03-01']
eventStrokeWidth 1.0 Width, in pixels, of the event lines.
eventLineColors ['#005a04', '#ccffbb', '#3a5f0b', '#005502'] Array of color values to use for the event line colors. If you list fewer colors here than you have lines in events, then the values will be cycled.




Donut (Doughnut) options
Option Default Description
data
(required) 		Example data: 
data: [
    {label: "Download Sales", value: 12},
    {label: "In-Store Sales", value: 30},
    {label: "Mail-Order Sales", value: 20}
]
 The data to plot. This is an array of objects, containing label and value attributes corresponding to the labels and sizes of the segments of the donut chart.
Note: by default, the segment with the greatest value will be initially selected. You can change the selection using the select(index) method on the object returned by Morris.Donut.
element
(required) 		The ID of (or a reference to) the element into which to insert the graph.
Note: this element must have a width and height defined in its styling.
colors ['#0B62A4', '#3980B5', '#679DC6', '#95BBD7', '#B0CCE1', '#095791', '#095085', '#083E67', '#052C48', '#042135'] An array of strings containing HTML-style hex colors for each of the donut segments. Note: if there are fewer colors than segments, the colors will cycle back to the start of the array when exhausted.
backgroundColor '#FFFFFF' This is actaully the color for the border of the dougnut sections.
labelColor '#000000' Specifies the color of the label in the center of the doughnut.
formatter Morris.commas A function that will translate a y-value into a label for the centre of the donut.
eg: currency function (y, data) { return '$' + y }
Note: if required, the method is also passed an optional second argument, which is the complete data row for the given value




Bar Chart Options
Option Default Description
Element
(required) 		The ID of (or a reference to) the element into which to insert the graph.
Note: this element must have a width and height defined in its styling.
data
(required) 		Example: 
	data: [
	    { y: '2006', a: 100, b: 90 },
	    { y: '2007', a: 75,  b: 65 },
	    { y: '2008', a: 50,  b: 40 },
	    { y: '2009', a: 75,  b: 65 },
	    { y: '2010', a: 50,  b: 40 },
	    { y: '2011', a: 75,  b: 65 },
	    { y: '2012', a: 100, b: 90 }
	  ]
The data to plot. This is an array of objects, containing x and y attributes as described by the xkey and ykeys options.
Note: the order in which you provide the data is the order in which the bars are displayed.
Note 2: if you need to update the plot, use the setData method on the object that Morris.Bar returns (the same as with line charts).
xkey
(required) 		Example: 'y' A string containing the name of the attribute that contains X labels.
ykeys
(required) 		Example: ykeys: ['a', 'b'] A list of strings containing names of attributes that contain Y values (one for each series of data to be plotted).
labels
(required) 		Example: labels: ['Series A', 'Series B'] A list of strings containing labels for the data series to be plotted (corresponding to the values in the ykeys option).
barSizeRatio 0.5 Percentage of the allocated space that a bar should take up.
barGap 3 Space in pixels between bars.
barColors ['#0b62a4', '#7a92a3', '#4da74d', '#afd8f8', '#edc240', '#cb4b4b' Array containing colors for the series bars.
stacked false Set to true to draw bars stacked vertically.
xLabelMargin 50 The space before and after xlables.




Line and Area Charts Options
Options Default Description
element
(required) 		The ID of (or a reference to) the element into which to insert the graph.
Note: this element must have a width and height defined in its styling.
data
(required) 		Example: 
	 data: [
	    { y: '2006', a: 100, b: 90 },
	    { y: '2007', a: 75,  b: 65 },
	    { y: '2008', a: 50,  b: 40 },
	    { y: '2009', a: 75,  b: 65 },
	    { y: '2010', a: 50,  b: 40 },
	    { y: '2011', a: 75,  b: 65 },
	    { y: '2012', a: 100, b: 90 }
	  ]
The data to plot. This is an array of objects, containing x and y attributes as described by the xkey and ykeys options.
Note: ordering doesn't matter here - you can supply the data in whatever order works best for you.
Note 2: if you need to update the plot, use the setData method on the object that Morris.Line returns. There's a setData example in the GitHub repo.
xkey Example: 'y' A string containing the name of the attribute that contains date (X) values. Timestamps are accepted in the form of millisecond timestamps (as returned by Date.getTime() or as strings in the following formats:
  • 2012
  • 2012 Q1
  • 2012 W1
  • 2012-02
  • 2012-02-24
  • 2012-02-24 15:00
  • 2012-02-24 15:00:00
  • 2012-02-24 15:00:00.000
Note: when using millisecond timestamps, it's recommended that you check out the dateFormat option.
Note 2: date/time strings can optionally contain a T between the date and time parts, and/or a Z suffix, for compatibility with ISO-8601 dates.
ykeys
(required) 		Example: ['a', 'b'] A list of strings containing names of attributes that contain Y values (one for each series of data to be plotted).
labels
(required) 		Example: ['Series A', 'Series B'] A list of strings containing labels for the data series to be plotted (corresponding to the values in the ykeys option).
lineWidth 3 Width of the series lines, in pixels.
pointSize 4 Diameter of the series points, in pixels.
lineColors ['#0b62a4', '#7A92A3', '#4da74d', '#afd8f8', '#edc240', '#cb4b4b', '#9440ed'] Array containing colors for the series lines/points.
pointWidths [1] This is to give different series different point sizes.
pointStrokeColors ['#ffffff'] Colors for the outlines of the series points. (#ffffff by default).
pointFillColors [] Colors for the series points. By default uses the same values as lineColors
smooth true Set to false to disable line smoothing.
xLabels 'auto' Sets the x axis labelling interval. By default the interval will be automatically computed. The following are valid interval strings:
  • "decade"
  • "year"
  • "month"
  • "day"
  • "hour"
  • "30min"
  • "15min"
  • "10min"
  • "5min"
  • "minute"
  • "30sec"
  • "15sec"
  • "10sec"
  • "5sec"
  • "second"
xLabelFormat null A function that accepts Date objects and formats them for display as x-axis labels. Overrides the default formatter chosen by the automatic labeller or the xLabels option.
eg: function (x) { return x.toString(); }
continuousLine true When set to false (the default), all null and undefined values in a data series will be ignored and lines will be drawn spanning them.
When set to true, null values will break the line and undefined values will be spanned.
Note: in v0.5.0, this setting will be removed and the behaviour will be to break lines at null values.






Area Charts only Options
Option Default Description
fillOpacity 'auto' Change the opacity of the area fill colour. Accepts values between 0.0 (for completely transparent) and 1.0 (for completely opaque).
behaveLikeLine false Set to true to overlay the areas on top of each other instead of stacking them.
